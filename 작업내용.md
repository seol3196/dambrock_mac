## Firebase 현재 상태

Authentication:

- 이메일/비밀번호 로그인은 사용 설정됨
- `admin@damvyeorak.local` Auth 계정은 생성됨
- UID: `x2a0wZpcs3gsmlvf8qf2VhWeG2G3`

Firestore:

- Firestore Database는 생성됨
- admin 권한 문서는 아직 콘솔에서 직접 만들어야 함
- 이유: 현재 Firestore 규칙 때문에 클라이언트 SDK의 `users/{uid}` 문서 쓰기가 `PERMISSION_DENIED`로 막힘

## 다른 PC에서 실행

```bash
npm install
cp .env.example .env
npm run dev
```

`.env`는 Git에 올리지 않았습니다. Firebase 콘솔의 웹 앱 설정값을 다시 넣어야 합니다.

필요한 값:

```text
VITE_FIREBASE_API_KEY
VITE_FIREBASE_AUTH_DOMAIN
VITE_FIREBASE_PROJECT_ID
VITE_FIREBASE_STORAGE_BUCKET
VITE_FIREBASE_MESSAGING_SENDER_ID
VITE_FIREBASE_APP_ID
```

## Git 상태

- 독립 저장소 위치: `C:\dev\담벼락\.git`
- 브랜치: `main`
